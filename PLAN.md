# 個人記帳頁面 — 每日趨勢圖 UI/UX Review & 優化計畫

## 現有實作分析

### 目前架構
- **圖表庫**: Chart.js (bar chart)
- **資料來源**: `PersonalExpenseSummaryResponse.dailyAmounts` (Map<LocalDate, BigDecimal>)
- **容器**: 固定高度 160px canvas，水平捲動 (`overflow-x-auto`)
- **切換**: 類別分布 (doughnut) / 每日趨勢 (bar) 兩個 tab

### 截圖觀察到的問題

| # | 問題 | 嚴重度 | 類別 |
|---|------|--------|------|
| 1 | **Bar chart 用於時間序列數據** — 業界標準是 Line/Area chart 用於趨勢，Bar chart 用於分類比較 | HIGH | 圖表類型 |
| 2 | **X 軸日期標籤旋轉 45°** — 佔用大量垂直空間，閱讀困難，mobile 更糟 | HIGH | 可讀性 |
| 3 | **右半空間完全浪費** — 無數據的日期 (2/14-2/19) 仍顯示完整空白欄位 | HIGH | 空間利用 |
| 4 | **Y 軸數字 (20,000) 無貨幣符號** — 用戶無法立即辨識數值含義 | MEDIUM | 資訊呈現 |
| 5 | **無 tooltip** — 無法看到精確金額，只能目測 | MEDIUM | 互動性 |
| 6 | **全部同色 (#F97316 橘色)** — 無法區分有支出/無支出日，也無預算參考線 | MEDIUM | 視覺層次 |
| 7 | **固定 responsive:false** — 不會自適應容器大小 | MEDIUM | 響應式 |
| 8 | **canvas minWidth = size*44** — 粗暴計算，日期少時太窄，多時需大量捲動 | LOW | 佈局 |

## 優化建議方案

### 方案 A: 漸進優化 (改善現有 Bar Chart) — 推薦

保留 bar chart 但大幅改善呈現方式，改動最小、風險最低。

#### A1. X 軸標籤優化
```
現在: 2026-02-11  2026-02-12  2026-02-13  (旋轉45°, 完整日期)
優化: 2/11  2/12  2/13  2/14  ...         (水平, 簡短格式)
```
- 移除年份，只顯示 `M/d` 格式
- 標籤水平排列 (不旋轉)
- 當日期超過 14 天，每隔一天顯示標籤

#### A2. 只顯示有意義的日期範圍
- 起點: 行程開始日 或 第一筆消費日 (取較早者)
- 終點: 今天 或 行程結束日 (取較早者)
- 移除完全無意義的未來空白日期

#### A3. Y 軸加入貨幣格式
```
現在: 20,000 / 10,000 / 0
優化: $20K / $10K / $0
```

#### A4. 加入 Tooltip
- hover/tap 時顯示: 日期 + 精確金額
- 格式: `2/13 — $5,200`

#### A5. 視覺增強
- 有支出的日期: 橘色 bar (#F97316)
- 無支出的日期: 不顯示 bar (gap)
- 加入日均線: 灰色虛線水平標記

#### A6. 響應式改善
- `responsive: true` + `maintainAspectRatio: false`
- 容器寬度自適應，短行程不需捲動
- 長行程 (>14天) 保留水平捲動

### 方案 B: 升級為 Line/Area Chart

將 bar chart 替換為 area chart，更符合「趨勢」語意。

- 類型: Area chart (填充 20% opacity)
- 線條: 平滑曲線 (tension: 0.3)
- 數據點: 圓點 + hover 放大
- 填充: 漸層色 (橘色到透明)
- 趨勢感更強，但需要 3+ 數據點才有意義

### 方案 C: 雙模式 (少數據 Bar / 多數據 Line)

智能切換:
- ≤ 7 天: 使用 bar chart (離散數據更直觀)
- > 7 天: 自動切換為 area chart (趨勢更清晰)

## 推薦方案: A (漸進優化)

### 理由
1. **風險最低** — 不改變圖表類型，只優化呈現
2. **一致性** — 與類別分布 (doughnut) 搭配協調
3. **行程多數 ≤ 14 天** — bar chart 對短期離散數據其實很適合
4. **開發效率** — 只改 JS 配置，不需重寫模板

### 實作步驟

#### Phase 1: Chart.js 配置優化 (personal-expense.js)

改動範圍: `initPersonalCharts()` 函數中 daily bar chart 部分 (L377-400)

1. **X 軸**: `M/d` 簡短格式 + 水平排列 + 超過 14 天自動間隔顯示
2. **Y 軸**: 加入貨幣前綴 `$` + K 縮寫 + `beginAtZero: true`
3. **Tooltip**: 自訂 callback 顯示格式化金額
4. **響應式**: `responsive: true`, 移除 `responsive: false`
5. **日均線**: Chart.js annotation plugin 或 dataset 方式

#### Phase 2: 日期範圍裁剪 (personal-expense.js)

1. 過濾掉尚未到來的未來空白日期
2. 保留到「今天」或行程結束日為止

#### Phase 3: 視覺微調

1. Bar 圓角: `borderRadius: 6` (現有 4 → 6)
2. Bar 最大寬度: `maxBarThickness: 32`
3. Grid line: Y 軸虛線 + X 軸隱藏
4. 日均參考線: 灰色虛線水平標記

#### Phase 4: 模板微調 (personal-tab.html)

1. 移除 `data-min-width` 硬算邏輯
2. 容器改為自適應 + 條件捲動
